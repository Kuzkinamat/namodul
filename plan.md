# План переработки стратегии и управления

## Цель
Улучшить логику расчета точек входа и интерфейс управления стратегией, чтобы пользователь мог задавать сложные условия на JavaScript с доступом к истории индикаторов, ценам и статистике сделок. Доработать панель настроек для удобства использования, унифицировать стили кнопок, добавить кнопку Apply для перерисовки индикаторов.

## Основные изменения

### 1. Выделение расчёта стратегии в отдельный скрипт
- Создать `strategy-core.js`, содержащий чистую логику расчёта сигналов без зависимостей от UI.
- Перенести функции: `createConditionContext`, `evaluateCondition`, `calculateSignals`, `calculatePnL`, `applyIndicatorSettings`, `applyStrategySettings`.
- Оставить в `strategy.js` только интеграцию с UI (отображение маркеров, вызовы из интерфейса).

### 2. Добавление кнопки Apply под окном настроек индикаторов
- В модальном окне настроек индикаторов (id="indicator-settings-modal") добавить кнопку "Применить" (Apply) рядом с существующими кнопками "Отмена" и "Применить".
- Кнопка должна применять настройки индикаторов и немедленно перерисовывать индикаторы на графике без закрытия модального окна (или с закрытием, по выбору).
- Альтернативно, добавить кнопку "Apply" в панели настроек (не модальное окно) для быстрого применения изменений параметров индикаторов.

### 3. Реорганизация интерфейса: перенос запуска стратегии, кнопки SETTINGS, экономия места
- **Перенос запуска стратегии**: кнопка "▶ Run Strategy" вынесена из выпадающего меню "STRATEGY ▾" на панель настроек справа от кнопки Apply (или в отдельную панель управления). Это позволит запускать стратегию одним кликом без раскрытия меню.
- **Перенос кнопки SETTINGS**: убрать кнопку "SETTINGS" из топбара (строка 455). Вместо неё добавить чекбокс "Показать настройки" в выпадающем меню стратегии (или отдельный пункт "Settings"), который будет открывать/закрывать панель настроек.
- **Экономия места**:
  - Убрать лишние рамки и границы у панелей, уменьшить отступы.
  - Сократить высоту топбара с 45px до 40px.
  - Уменьшить размер шрифта кнопок с 11px до 10px.
  - Сжать марджины между элементами топбара.
  - Убрать дублирующиеся разделители в выпадающих меню.
  - Объединить меню "INDICATORS" и "STRATEGY" в одно меню "ANALYSIS" для экономии горизонтального пространства.
- **Результат**: более компактный интерфейс, больше места для графика.

### 4. Унификация стилей кнопок
- Создать единый базовый стиль для всех кнопок (`.btn`) с общими свойствами: шрифт, цвет фона, цвет текста, отступы, скругления.
- Модификаторы для разных типов кнопок: `.btn-primary` (основные действия), `.btn-secondary` (второстепенные), `.btn-danger` (опасные).
- Применить к классам: `.nav-btn`, `.settings-btn`, `.btn-apply`, `.btn-cancel`, `.modal-actions button`.
- Упростить CSS, сократив дублирование.

### 5. Редизайн панели настроек с вкладками
- Разделить панель настроек на вкладки: "Условия", "Индикаторы", "Экспорт".
- Вкладка "Условия": редакторы для buy if и sell if с улучшенным интерфейсом.
- Вкладка "Индикаторы": заменить JSON-редактор на форму с полями ввода для каждого параметра (период SMA, период BB, стандартное отклонение, параметры MACD, Stochastic).
- Вкладка "Экспорт": кнопки экспорта данных в CSV/JSON.
- Добавить кнопку "Применить все настройки" (Apply All) для применения изменений из всех вкладок.

### 6. Улучшение редактора условий
- Добавить подсказки (автодополнение) для переменных и функций при вводе в текстовые поля условий.
- Создать справочник переменных с описанием, доступный по нажатию на иконку "?".
- Добавить кнопки быстрой вставки часто используемых условий (как уже есть, но улучшить визуально).
- Возможность проверки условия на текущей свече (кнопка "Проверить условие").

### 7. Замена JSON-редактора на форму с полями ввода
- Убрать текстовое поле `indicator-settings` (JSON) и заменить его на отдельные поля для каждого индикатора.
- Группировать по индикаторам: SMA (период), Bollinger Bands (период, stdDev), MACD (fast, slow, signal), Stochastic (K, D, slowing).
- Добавить чекбоксы для включения/отключения индикаторов.

### 8. Интеграция изменений в основной код
- Обновить `main.js` для работы с новой панелью настроек (обработка вкладок, применение настроек).
- Обновить `strategy.js` для использования `strategy-core.js`.
- Обеспечить обратную совместимость: старые настройки должны работать (миграция JSON в форму).

### 9. Тестирование и проверка работоспособности
- Проверить, что все кнопки работают, индикаторы перерисовываются, условия применяются.
- Убедиться, что стили кнопок едины и интерфейс не сломан.
- Протестировать на разных данных (локальных и от API).

## Этапы реализации

1. **Создание strategy-core.js** – выделение ядра стратегии.
2. **Добавление кнопки Apply** в модальное окно настроек индикаторов.
3. **Реорганизация интерфейса** – перенос запуска стратегии, кнопки SETTINGS, экономия места.
4. **Унификация стилей кнопок** – рефакторинг CSS.
5. **Редизайн панели настроек** – разметка вкладок, формы.
6. **Улучшение редактора условий** – добавление подсказок и справочника.
7. **Замена JSON-редактора** – создание формы ввода параметров.
8. **Интеграция** – обновление main.js и strategy.js.
9. **Тестирование** – проверка всех функций.

## Временные оценки
- Выделение strategy-core.js: 1 час.
- Кнопка Apply: 0.5 часа.
- Реорганизация интерфейса: 1-2 часа.
- Унификация стилей: 1 час.
- Редизайн панели настроек: 2-3 часа.
- Улучшение редактора условий: 1-2 часа.
- Замена JSON-редактора: 1 час.
- Интеграция: 1-2 часа.
- Тестирование: 1 час.
- Итого: 9-13 часов.

## Риски
- **Сложность интеграции**: изменения могут затронуть много файлов, нужна аккуратность.
- **Производительность**: добавление множества полей ввода может замедлить рендеринг панели настроек.
- **Обратная совместимость**: пользователи, хранящие настройки в JSON, могут потерять их при переходе на форму.
