# **Инструкция для разработки терминала/бота проверки торговых стратегий** {#instrukciya-dlya-razrabotki-terminala/bota-proverki-torgovyh-strategij}

## **Технологический стек** {#tehnologicheskij-stek}

* HTML, JavaScript, Python

* Библиотека TradingView Lightweight Charts v5.1.0 (использовать синтаксис именно этой версии)

## **Правила работы с кодом** {#pravila-raboty-s-kodom}

### **1. Процесс разработки** {#1-process-razrabotki}

* Написание/правка кода выполняется только по команде "давай", после уточнения деталей задачи

* Не изменять код, не имеющий отношения к текущей задаче

* При добавлении нового функционала обязательно добавлять вывод логов для отладки:

  * Ответы сервера

  * Фрагменты данных

  * Ключевые этапы выполнения

### **2. Архитектурные принципы** {#2-arhitekturnye-principy}

* Соблюдать модульность и разделение ответственности

* Использовать чистые функции там, где это возможно

* Обеспечивать обработку ошибок на всех уровнях

* Документировать публичные API и сложную логику

## **Функциональные требования** {#funkcionalnye-trebovaniya}

### **Итоговый функционал системы:** {#itogovyj-funkcional-sistemy}

#### **1. Выбор рабочего диапазона** {#1-vybor-rabochego-diapazona}

* Интерфейс для выбора временного диапазона данных

* Поддержка различных таймфреймов (минуты, часы, дни)

* Возможность задания пользовательских диапазонов

#### **2. Выбор источника данных** {#2-vybor-istochnika-dannyh}

* Поддержка различных источников данных (CSV файлы, API провайдеры)

* Модульная архитектура для легкого добавления новых источников

* Конфигурация подключения к источникам

#### **3. Подгрузка пар/файлов/тикеров** {#3-podgruzka-par/fajlov/tikerov}

* Динамическая загрузка доступных торговых пар

* Поддержка загрузки из файловых источников

* Кэширование списка тикеров для производительности

#### **4. Выбор пары** {#4-vybor-pary}

* Интерфейс выбора конкретной торговой пары

* Фильтрация и поиск по списку пар

* Отображение дополнительной информации о выбранной паре

#### **5. Загрузка данных** {#5-zagruzka-dannyh}

* Загрузка исторических данных в соответствии с выбранным диапазоном

* Оптимизация запросов к источникам данных

* Валидация и нормализация полученных данных

* Обработка пропущенных данных и выбросов

#### **6. Обновление данных онлайн** {#6-obnovlenie-dannyh-onlajn}

* Реализация механизма реального времени

* Поддержка WebSocket или polling для обновлений

* Обработка новых данных и обновление графика

* Управление частотой обновлений

## **Требования к реализации** {#trebovaniya-k-realizacii}

### **Для TradingView Lightweight Charts v5.1.0:** {#dlya-tradingview-lightweight-charts-v510}

* Использовать API именно версии 5.1.0

* Следовать документации и примерам для этой версии

* Учитывать особенности и ограничения данной версии

### **Логирование и отладка:** {#logirovanie-i-otladka}

* Все ключевые операции должны логироваться

* Логи должны содержать достаточно информации для диагностики

* Реализовать уровни логирования (debug, info, error)

### **Производительность:** {#proizvoditelnost}

* Оптимизировать загрузку и отрисовку больших объемов данных

* Использовать lazy loading где это возможно

* Минимизировать перерисовки графиков

## **Процесс разработки** {#process-razrabotki}

1. **Получение задачи**: Уточнение деталей и требований

2. **Планирование**: Определение необходимых изменений

3. **Реализация**: Написание кода с соблюдением правил

4. **Тестирование**: Проверка функциональности и логирование

5. **Документирование**: Обновление документации при необходимости

## **Важные замечания** {#vazhnye-zamechaniya}

* Все изменения должны быть инкрементальными и обратно совместимыми

* При внесении изменений в существующий код обеспечить backward compatibility

* Тестировать новые функции на различных наборах данных
